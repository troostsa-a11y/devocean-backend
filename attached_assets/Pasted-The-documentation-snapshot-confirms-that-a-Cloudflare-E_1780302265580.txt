The documentation snapshot confirms that a Cloudflare Error 1101 explicitly means a Worker threw a JavaScript exception at runtime. Since your home route serves a static index.html file via client-side rendering (CSR), a standard user visit will not hit the V8 execution environment or trigger this code.If the 1101 exception is appearing on a page load, the site is not behaving as a pure static asset distribution.Why You Are Seeing an Error 1101 on a CSR SiteIf a visitor hits your URL and receives an Error 1101, it points to one of three edge configuration routing issues:Top-Level Pages Functions Crash: You mentioned having two contact endpoints (/api/contact and /api/experience-inquiry). If you have a root _middleware.js file or an unhandled global export at the top of your functions/ directory, Cloudflare will try to compile it for every single incoming request. A top-level syntax error, an invalid runtime hook, or a broken third-party module import in that folder will crash the edge router instantly.Missing System Environment Variables: Even if your try/catch block safely handles a missing API token, referencing an undeclared environment binding globally in the top-level outer scope (e.g., const client = new Resend(env.RESEND_API_KEY) placed outside the onRequest handler) will throw an unhandled reference exception during worker script startup.Legacy Cloudflare Workers Routes Overlap: If there is an old, active Cloudflare Worker associated with the same custom domain (://devoceanlodge.com*), that independent script will intercept traffic before it ever touches your new Pages deployment.
